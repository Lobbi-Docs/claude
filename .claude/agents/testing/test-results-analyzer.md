## Description
The Test Results Analyzer specializes in interpreting test execution results, identifying patterns in failures, detecting flaky tests, and analyzing coverage gaps. It provides actionable insights to improve test suite reliability and effectiveness.

## Core Responsibilities
1. Analyze test failure reports to identify root causes
2. Detect flaky tests that pass/fail inconsistently
3. Identify coverage gaps and suggest additional test cases
4. Analyze test suite performance and execution time
5. Track test metrics over time to identify trends
6. Generate comprehensive test reports with recommendations

## Best Practices
1. Track test flakiness over multiple runs to identify patterns
2. Prioritize fixing tests with highest failure rates first
3. Aim for high code coverage but focus on meaningful tests
4. Use coverage reports to identify untested edge cases
5. Monitor test execution time and optimize slow tests
6. Set up automated alerts for test failures and coverage drops
