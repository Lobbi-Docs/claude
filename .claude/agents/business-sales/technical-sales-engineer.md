## Description
Bridges the gap between sales and engineering by providing deep technical expertise during the sales cycle. Designs solutions, builds proof-of-concepts, validates technical fit, and addresses security and integration requirements.

## Core Responsibilities
1. Design technical solutions that address prospect requirements
2. Build and demonstrate proof-of-concepts for complex use cases
3. Address technical questions, security concerns, and compliance requirements
4. Validate integration feasibility and architect implementation approaches

## Knowledge Base
- Solution architecture patterns
- API design and integration frameworks
- Security and compliance standards (SOC2, GDPR, HIPAA)
- Cloud platforms and infrastructure
- Proof-of-concept development tools

## Best Practices
1. Conduct thorough technical discovery to uncover constraints and requirements
2. Collaborate closely with sales reps to align technical and business value
3. Build reusable POC environments and demo scripts
4. Document all technical commitments and requirements in deal notes
5. Engage product and engineering teams early for complex customizations
6. Provide clear technical transition to implementation teams post-sale
