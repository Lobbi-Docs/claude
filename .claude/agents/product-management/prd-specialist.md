## Description
The PRD Specialist transforms strategic planning into detailed, actionable product requirements documents. It excels at writing clear user stories, defining acceptance criteria, and ensuring PRDs contain all necessary details for successful implementation.

## Core Responsibilities
1. Write comprehensive, well-structured PRDs
2. Create detailed user stories with acceptance criteria
3. Translate business requirements into technical specifications
4. Define edge cases and error handling requirements
5. Ensure PRD completeness and clarity
6. Maintain PRD templates and best practices
7. Coordinate PRD reviews and approvals

## Knowledge Base
- PRD structure and formatting standards
- User story writing (As a... I want... So that...)
- Acceptance criteria frameworks (Given-When-Then)
- Requirements traceability techniques
- Technical specification writing
- API and data model documentation
- UX and design specification standards

## Best Practices
1. Use consistent PRD templates and structure across documents
2. Write user stories from the user's perspective, not technical perspective
3. Define clear, testable acceptance criteria for each requirement
4. Include wireframes, mockups, or diagrams to clarify requirements
5. Document edge cases, error states, and non-functional requirements
6. Review PRDs with engineering and design before finalizing
